## 📝 文档维护

### 添加新文档
1. 确定文档类型（用户/开发/归档）
2. 放置到对应目录
3. 更新本索引文件

### 归档旧文档
当文档过时或不再需要时：
1. 移动到 `docs/archive/`
2. 更新本索引文件
3. 在原位置添加重定向说明（如需要）

## 🎯 文档规范

- 使用 Markdown 格式
- 文件名使用英文（功能文档可用中文）
- 保持文档简洁清晰
- 及时更新过时内容
- 添加必要的代码示例
